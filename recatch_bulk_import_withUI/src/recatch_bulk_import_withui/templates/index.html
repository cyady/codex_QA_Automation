LOCAL OPS UI

Re:catch Bulk Import

비즈니스 팀원이 각자 로컬에서 실행하는 전용 UI입니다. 실행은 localhost에서만 이뤄지며, CSV 범위 선택, 필드 매핑, 진행률 확인, 재시작 권장 범위를 한 화면에서 관리합니다.

운영 원칙
  • 필드명은 Re:catch 화면의 텍스트와 정확히 일치해야 합니다.
  • 작업 중 브라우저를 닫아도 runner는 재시도할 수 있습니다.
  • Vibium/chromedriver 충돌은 자동 정리 루틴이 먼저 처리합니다.
  • Agentation은 `localhost:4747` MCP endpoint를 사용합니다.

실행 설정

먼저 CSV 경로와 파트 범위를 확인한 뒤 헤더를 불러오세요.

원본 CSV를 입력하면 split 파일은 위 디렉터리에 저장됩니다. 이미 split CSV가 있으면 원본 CSV는 비워둘 수 있습니다.
현재 env 기준 기본값이 로드되어 있습니다.

필드 매핑

정확한 필드명 입력이 필요합니다. 검색어와 선택지 텍스트를 각각 입력하세요.

필드명은 Re:catch UI에 보이는 텍스트와 정확히 일치해야 합니다. 잘못 입력하면 자동화가 실패하거나 다른 필드가 선택될 수 있습니다.
헤더를 불러오면 여기에 CSV 헤더 정보와 split 파일 안내가 표시됩니다.
CSV 헤더 검색어 선택지 텍스트 Skip
먼저 "CSV 헤더 불러오기"를 눌러 헤더를 확인하세요.
