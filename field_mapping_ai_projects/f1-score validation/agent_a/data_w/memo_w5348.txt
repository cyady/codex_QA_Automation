의제





공동주택(아파트 단지) 짐 운반 카트 로봇 서비스 도입 검토



기술적 난관 (엘리베이터 연동, 보안 등) 논의



실증 사업 예산 및 역할 분담 방식 협의



내년 실증 일정 및 향후 다단지 확산 전략 논의

미팅 내용

1. 배경





기존 공동주택 로봇 실증(순찰·배송 등)은 많았으나 입주민 체감 효용 및 지속성 부족



입주민 입장에서 가장 직관적인 가치: “짐 운반 카트 로봇”





도입 구간 : 장을 보고 나서 짐 옮기기 (지하주차장-->세대)



확보된 예산 : 3,000만원 내외





해당 실증 사업 확정 후, 여러 단지에 확대 적용 및 포스코 계열에 소개



AMR 유형 : 롤테이너형 AMR 1대



제안된 시나리오





입주민 앱/월패드로 로봇 호출 → 지하주차장 지정 위치로 로봇 도착



짐 적재 후 세대 정보(예: 101동 101호) 기반 자율주행



엘리베이터 연동으로 세대 현관 앞까지 이동 → 도착 알림 후 짐 수령



복귀 버튼/앱 명령으로 로봇 자동 복귀(홈스테이션)



필요 시 사람 추종 모드(입주민이 걷고 로봇이 따라오는 카트 모드)도 옵션 검토

2. 엘리베이터/출입문 연동





공동 현관문, 엘리베이터, 자동문 등 공용부 설비와의 연동 필수



현실 구조:





엘리베이터와의 직접 연동은 홈넷 및 엘리베이터 업체와 협의 요망



로봇사는 스마트홈/홈넷 플랫폼과 API 연동하는 구조가 적합



필요 신호: 층 호출, 도착, 문열림/문닫힘, 혼잡 상태 등



사람과 동승 시:





출퇴근 등 혼잡 시간에는 대기/우회 기능 필수



비전 기반으로 탑승 가능 공간 판단 후 여유 있을 때만 탑승

3. 지하주차장





지하주차장은 차량 이동, 교차로, 시야 제한 등으로 난이도 가장 높음



맵 구성:





B3, 1층, 동&층별 맵을 관제에 등록



구역이 넓을 경우 로봇 역할 분할





A: 지하주차장 / B: 1층 / C: 세대층(엘리베이터 릴레이)





차량 시야:





로봇 높이가 낮아 시야에서 사라질 수 있음 →
상부 플래그, 경광등, 방향지시등, 교차로 접근 시 자동 점멸 로직 등 검토



어린이/입주민 상호작용:





로봇을 막거나 버튼을 누르는 상황 대비 필요



인체 인식 기반으로 일정 시간 방해 시





TTS로 “제가 지나가야 해요” 등 안내



다중 경로 확보로 우회 가능하게 설계

4. 음영 지역 대응





기본 통신: 단지 WiFi



지하주차장 등 WiFi 커버 안 될 시 LTE/5G 모듈 탑재 검토



구조:





“로봇 ↔ 단지 내 로봇/스마트홈 서버 ↔ 관제 서버” 구조 설계





"각 세대 월패드 ↔ 단지 서버 ↔ 클라우드(플랫폼) 구조와 동일



음영지역 :





일정 시간 통신 두절 시 정지 → 홈스테이션 자동 복귀



복귀 로그로 음영 구역 확인 후 동선 재설계(우회 경로)

5. 비정형 공간 및 무게 (방지턱, 문지방, 경사, 하중)





검토





10~15° 경사 + 50kg 적재 주행 가능 예상



병원 레퍼런스 기준 2.5cm 문지방·턱 통과 경험



공동주택 환경:





지하 방지턱, 자동문 단차(2.8cm 등), 점자블록, 엘리베이터 문턱 등



실제 대상 현장(천우룡 등)에서 반복 테스트 필수



하중





장바구니 속 등 고려해





액티브 서스펜션 + 버퍼 센서로 수평 유지



방진·방수·카메라/센서 보호 설계 강화 필요

6. 네플러





클라우드 선호



플랫폼 구조:





단지별 별도 시스템보다는





하나의 대형 Nepler 내에 단지별 프로젝트를 논리 분리하는 방향 선호



건설사, 스마트홈 플랫폼(포스코 DX), 홈넷, 엘리베이터 업체 간 역할/보안 정책은 별도 협의 필요
