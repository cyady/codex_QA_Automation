미팅 내용

1. 배터리/충전





25Ah → 42Ah 배터리, 6A → 15A 충전기로 변경 검토.



실제 사용 기준 1시간에 약 20% 소모, 충전시간 과다 문제 → 고속 충전 및 절전모드 필요.



배터리 보증 6개월, 제품 나머지는 1년.



2. 주행·API/HMI





현재 직각·정지 위주의 물류용 무빙이 도서관 환경에 거칠게 보임.



서빙 로봇과 같은 부드러운 곡선 주행으로 변경 가능(알고리즘 전환 필요).



크루즈(순회) 모드, 목적지 대기시간, 속도 제어, 위치/상태 이벤트 푸시 등 API 확장 요구.



맵 편집/이관 툴과 도면-맵 매칭(좌표 변환)에 대한 지원 및/또는 유상 지원 검토.



3. 인증





42Ah 배터리 UN38.3 1분기 내 확보 예정, 추가 인증(ICE 등)은 예산 검토 중.



로봇 완제품 KC 인증으로 공공기관 납품에 문제 없도록 방향 정리.



KC/BF 인증용 시험에서 모터·센서 연속 동작 확인용 API/펌웨어 필요.

4. 생산/리드타임/OEM





부품 리드타임 포함 납기 약 8주로 보는 것이 현실적, 현재 재고는 거의 없음.



하드웨어 커스텀 시 최소 수량(예: 20대 수준) 필요 가능성, 표준 사양은 MOQ 제한 최소화.



상부 모듈 부품을 Polaris3D가 제공하고, 완제품 조립·납품은 로봇사 측에서 수행하는 OEM 구조 가능.

5. 유지보수/AS





현재는 개발팀 직접 대응 단계, 향후에는 “도서관 → 지역 협력업체/CS → 로봇사” 3단계 구조로 정리 예정.



원격 진단 필수, 원거리(제주 등) 현장은 파트너사가 1차 대응 후 필요 시 로봇사 지원.



협력업체 기술 교육·기술이전 필요.

6. 호출벨·관제·위치추적





서빙 로봇용 호출벨을 AMR에 적용 가능하나, 수신기 위치/감도 검증과 일부 커스텀 필요.



호출 상태·배터리 상태 등은 관제 시스템과 연동해 한 화면에서 모니터링하는 구조 지향.



로봇 좌표를 도면에 매핑해 사서가 로봇 위치를 직관적으로 볼 수 있도록 구현 필요.

7. MOU/파트너십





양사 공동으로 도서관 로봇 시장 개척을 대외적으로 알릴 수 있는 MOU에 긍정적 합의.
