1. 배경 및 회사 소개
대표님은 1993년 창업, 30년 넘게 건축 기계 설비 분야 소프트웨어 개발 및 영업을 진행
주요 제품: 꼬매(구조 설계 프로그램), 화이어(소방 설계 프로그램) 등 자체 개발 솔루션
영업은 직원보다는 대표님이 직접 수행, 주로 입소문·DM 발송으로 이뤄짐
기존 내부 시스템은 엑셀 기반을 발전시킨 자체 프로그램: 거래처 등록 → 견적서 발송 → 거래명세서 전환 → 간단 통계
2. 현재 문제점
전화/영업 대응 시 이력 관리 부재: 동일한 고객 응대 중복, 의사소통 단절
데이터 중복 입력: 회사명 변경, 직원 교체 시 기존 고객을 새로 등록하는 문제 발생
체계적 관리 부족으로 고객 DB 활용 한계
3. CRM 도입 필요성
히스토리 관리: 누가 전화를 받든 동일한 고객 정보를 즉시 확인 가능
업그레이드·크로스셀 기회 발굴: 기존 버전 고객 대상 DM 발송 및 기능 홍보
일관성 있는 고객 관리로 매출 확대 기대
4. 리캐치 소개
DB 이관 방법: (1) 업체 의뢰(비용 발생 가능), (2) 직접 CSV/엑셀 업로드
이메일 캠페인 기능: DB 기반 그룹 필터링 → 수신자 그룹 생성 → 직접 이메일 발송 (메일플러그 사용 불필요)
광고성 이메일은 법적으로 ‘광고’ 표시 필요
정보성 콘텐츠(예: 설계 동향 리포트) 활용 시 부담 완화 가능
영업 파이프라인 관리: 기본 단계(진행 전/중, 성공, 실패) + 커스터마이징 가능
제품·버전별 필드 생성 가능: 꼬매/화이어, 버전(93년판 ~ 2026판) 관리 가능
리서치 AI 기능: 나이스 신용평가 및 언론 기사 연동 → 거래처 재무/현안 파악
대시보드 통계: 파이프라인 기반 매출 추정, 수주 목표 대비 진행 현황 확인
웹폼(리드 라우터): 홈페이지에 삽입해 고객 문의 자동 DB화, 담당자 자동 배정
지원: 채널톡 상담, 1회 온라인 온보딩 제공. 원격 지원은 별도 비용
5. 대표님의 질문/우려
동일 회사 내 여러 부서 요청 발생 시 CRM에서 어떻게 구분·기록하는지 → 부서별 딜 생성 가능
매출 예측 자동화 여부 → 예측은 불가, 다만 파이프라인 기반 수주 금액 통계 제공
세일즈/마케팅 패키지 차이 → 마케팅 패키지 구매 시 세일즈 기능 포함
데이터 연동 여부 → 세일즈만 구매 시 웹폼 연동 불가, 마케팅 포함해야 가능
커스터마이징 가능 여부 → 로드맵과 방향성 맞을 시 일부 가능, 별도 용역 형태
회사 안정성/지속성 → 2020년 창업, 누적 투자 130억, 5년차, 직원 30명대
6. 핵심 요약
DCS는 30년 업력의 설비 소프트웨어 개발사로 자체 프로그램 운영 중이나 CRM 체계 부재
대표님의 주요 관심사는 고객 히스토리 관리·DB 기반 매출 확대·업그레이드 마케팅
리캐치 CRM은 이를 지원하며, 특히 DB 이관, 이메일 캠페인, 영업 파이프라인, 웹폼 연동, AI 기반 고객 정보 확인 등이 강점
다만 견적서 자동 작성·관리 기능은 제공하지 않음
CRM 도입 성공을 위해서는 팀원들의 적극적 입력·사용 문화 정착이 중요
7. 계획
추석이후로 최소플랜으로 유료 트라이얼 시작 여부 결정할 것
시작하게 된다면 다른 팀원들과 살펴보고 계속 사용 결정할 것
