킹콩F&B와 별도로 신규 법인 '킹콩애드(마케팅 대행사)'를 설립했는데 애드에 대한 관리 필요성을 느낌



비즈니스 모델: 소규모 광고 대행사(주요 타겟) 및 자영업자(2차 타겟)에게 바이럴 마케팅 상품 판매(블로그 후기, 영수증 리뷰 등)



CRM 활용 경험 있음: 플러그, 세일즈맵 사용 경험 있으나 UI 및 시스템 불만족으로 중단





뭐가 불만족이었냐?



1/ UI 및 데이터 가시성 문제





업체 등록 후 매일 결제가 발생하는 사업 구조인데, 기존 툴에서는 이러한 내용을 확인하기가 UI적으로 너무 답답했다고 함



2/ 수기 입력의 비효율성 (연동 부재)





자체 개발 중인 플랫폼에서 결제가 이루어질 때마다 CRM에 해당 내역을 일일이 기입해야 하는 방식이 킹콩애드의 시스템과 맞지 않다고 판단함



3/ 고객 유형별 관리의 어려움





주 고객인 '소규모 광고 대행사(B2B)'와 서브 고객인 '자영업자(B2C)'를 명확히 나누어 관리하는 것이 힘들었다고 함



"지금까지 국산 CRM 2개 다 우리가 원하는 기능 구현이 잘 안 되는데, 리캐치마저 어렵다고 하면 우리가 감안해야 하는 부분이라 생각한다"



c.f) 자체 웹 플랫폼 개발 중이며, 기존 카카오톡 위주의 수기 업무를 시스템화하려는 단계
