1/ 전사 통합 추진 핵심 배경 및 목적





현황) 현재 컨설팅 본부(IR/PR/IPO)와 디지털(DX) 본부가 각기 다른 방식으로 영업 및 고객 관리를 수행 중



문제점) 팀별/본부별로 리드 관리(시트), 미팅록(노션), 계약 관리(시트)가 분산되어 있어, 전사 관점의 통합 대시보드(매출, 현황) 확인이 불가능함



목표) 리캐치를 통해 영업~계약 데이터를 일원화하여 고객사별 히스토리(미팅, 계약 등)를 자산화하고, 전사적인 데이터 관리 체계를 구축





단순하게 최대한 많은 리소스( 데이터 기입 ) 들이지 않고 전사 관리 시스템을 구축하고 싶어함


2/ 업무 프로세스 설계 (Sales to Closing)
[As-Is] 기존 프로세스 (분절된 관리)





리드 접수: 다양한 경로 유입 → 구글챗 수동 공유 → 담당자 수동 배정.



영업 관리: 팀별 '구글 시트'에 리드 기록 → 유선/메일로 미팅 제안 → 구글 캘린더 생성.



미팅/협의: 미팅 진행 후 '노션(Notion)'에 미팅록 작성 (검색/통합 조회 어려움).



계약/매출: 계약 완료 시 구글챗 수동 공유 → 별도 '구글 시트'에 계약 정보 기록 → 외부 솔루션으로 세금계산서 발행.



사후 관리: 갱신/해지 시점 도래 시 다시 시트 확인 후 유선 안내.


[To-Be] 리캐치 도입 후 프로세스 (통합 및 자동화)





리드 접수: 리캐치 Form으로 접수 → Zapier 연동을 통해 구글챗 자동 알림.



영업 관리: 리캐치 내 '리드/딜' 생성 → 담당자 배정 및 파이프라인 이동 → 리캐치 캘린더/제안 기능 활용.



미팅/협의: 리캐치 내 메모/필드 기능을 활용해 미팅록 작성 (회사명 클릭 시 모든 히스토리 조회 가능하도록 구현).



계약/매출: 영업 성공 시 '계약 파이프라인'으로 딜 전환 → 구글챗 자동 알림(Webhooks) → 리캐치 내 계약 데이터 축적.



사후 관리: 리캐치 데이터 기반으로 갱신/해지 관리 및 이력 추적.
