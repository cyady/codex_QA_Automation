# 영림원 ERP-CRM 연동 개발 및 협업을 위한 전략 미팅록

## 1. 미팅의 전체 맥락
*   **참석자**: 리케치(Re:catch) 측 PM 및 대표, 영림원 파트너사(G2J) 대표
*   **미팅 목적**: 리케치 CRM과 영림원 ERP의 연동 방식, 프로젝트 진행 절차, 비용 구조 및 파트너십 모델에 대한 전문가 조언 확보
*   **전체적인 흐름**: 영림원 ERP의 제품군별 연동 가능성 검토 → 주요 타겟 고객사(드림칩, 위존) 현황 공유 → 연동 개발 비용 및 유지보수 정책 확인 → 향후 영업 협력 프로세스 정립

## 2. 모든 주요 논의 사항

### [영림원 ERP 제품군별 연동 특성]
영림원 ERP는 관리 주체와 제품 버전에 따라 연동 방식이 상이함이 논의되었습니다.
*   **K-System Ace (에이스)**: 가장 상위 버전의 구축형 제품. 연동 제한이 거의 없으며 DB-to-DB, API, 에이전트 설치 등 고객이 원하는 모든 방식의 연동이 가능함.
*   **System Cloud 4 (SCF)**: SaaS 형태의 클라우드 버전. 인프라가 클라우드에 있어 DB-to-DB는 권장되지 않으며, 표준 API를 통한 연동이 기본 원칙임.
*   **System Ever (에버)**: 가장 저가형 버전. 추가 개발 자체가 지원되지 않아 외부 시스템 연동이 매우 제한적임. "에버는 일단 연동이 안 된다고 보시는 게 맞다"는 전문가 의견이 있었음.

### [연동 주체 및 패키징 조건]
*   **연구소 주도 패키징**: 영림원 본사 연구소에서 공식 연동 패키지를 만들려면 최소 3~4개 이상의 고객사 레퍼런스가 확보되어야 함. 현재는 개별 연동으로 진행해야 하는 단계임.
*   **관리 주체 확인**: 고객사마다 관리하는 파트너사가 다르므로, 연동 논의 전 해당 고객사를 영림원 본사가 관리하는지, 특정 파트너사가 관리하는지 확인이 필수적임.

### [질의응답 요약]
*   **Q (리케치)**: "영림원 연구소와 직접 연동 개발을 진행할 수 있나요?"
*   **A (전문가)**: "연구소는 항상 바쁘기 때문에 새로운 스케줄을 바로 넣어주지 않습니다. 현재로서는 개별 연동을 진행해야 하며, 각 고객사를 관리하는 주체(파트너사 등)와 직접 소통해야 합니다."

## 3. 고객의 고민점과 니즈
*   **연동 경험 부족**: 리케치 측은 영림원 ERP와의 연동 프로젝트가 처음이라 구체적인 프로세스와 기술적 제약 사항에 대한 가이드가 절실한 상황임.
*   **고객사의 연동 요구**: 
    *   **드림칩(부품 유통사)**: 매출 600억 대의 강소기업으로, 재고 관리(SCM)와 주문 발주 시스템이 ERP와 연동되기를 강력히 원함.
    *   **위존**: 이미 리케치를 사용 중이며, 다음 단계(Next Phase)로 ERP 연동을 요구하고 있음.
*   **비용 산정의 어려움**: 고객사에게 연동 견적을 제시해야 하나, Man-day 단가나 유지보수 요율에 대한 기준이 없어 제안에 어려움을 겪고 있음.

## 4. 제품/서비스 제안 내용
*   **기술적 제안**: 클라우드 버전(SCF)의 경우 API 연동을 우선적으로 고려할 것을 제안함.
*   **비용 구조 제안**:
    *   **개발비**: 1 Man-day당 약 70만 원 수준으로 책정됨.
    *   **유지보수비**: 추가 개발된 연동 범위에 대해 개발비의 15%를 연간 유지보수료로 청구하는 것이 업계 표준임.
*   **영업 협력 모델**:
    *   **기존 ERP 고객**: 이미 파트너사가 지정되어 있으므로 리케치가 직접 연동 견적을 내거나 파트너사와 협업하여 진행.
    *   **신규 ERP 도입 고객**: 리케치가 영림원 ERP 도입을 소개할 경우, 이에 따른 영업 수수료(정산) 논의가 가능함.

## 5. 결정 사항과 액션 아이템
*   **고객사 ERP 버전 파악 (리케치 담당자)**: 오늘 예정된 드림칩 미팅에서 사용 중인 영림원 ERP 버전(Ace, SCF, Ever 중 무엇인지)과 사스(SaaS) 형태 여부를 반드시 확인할 것.
*   **고객사 관리 주체 확인 (G2J 대표)**: 리케치 측에서 드림칩과 위존의 사업자 정보를 공유하면, 영림원 내부망을 통해 현재 관리 파트너사가 어디인지 확인해 주기로 함.
*   **연동 견적 프로세스**: 구체적인 연동 범위(Scope)가 나오면 내부 개발팀과 협의하여 견적을 산출하기로 함. 리케치 측도 자사 API 개발 비용을 포함하여 고객에게 통합 견적을 제시할 수 있음.
*   **후속 연락 채널**: 영림원 고객가치팀의 영업 담당자(4명) 중 적합한 인물을 G2J 대표가 매칭하여 리케치 측에 소개해 주기로 합의함.
